# Compound Engineering Runtime

You are fixing a real Day.js bug in this repository.

Use a compact engineering loop:
1. Restate the observable failure and identify the smallest relevant code path.
2. Make the smallest source change that explains the failing benchmark test.
3. Run the focused Jest command from the task prompt.
4. Leave a concise note in your final answer with the changed file and command result.

Keep repo changes scoped. Do not change benchmark tests unless validation proves
the test patch itself is wrong.
